# 게임 기획 정리 (Phase 4)

## 한 줄 소개
Ordia식 슬링샷 이동을 베이스로, 앵커 그래프 위에서 적을 추격·사냥하는 **턴제 전략 액션 로그라이크**.

---

## 1. 조작
- Ordia식 **슬링샷**: 드래그로 조준 → 릴리즈로 발사.
- 신중함·정밀함이 필요한 이동. **조준 실패 시 의도치 않은 앵커로 이동** = 핵심 손맛/리스크.

## 2. 보드 구조
- 맵에 여러 **앵커**가 배치, **노드 그래프**로 관리.
- **1앵커 = 1점유** (플레이어 또는 적 1마리). 보드형.
- 자유 순회 가능. **도망도 정당한 전략**(생각할 시간 확보).

## 3. 턴 시스템
- **체스/바둑식 교대턴** (완전정보, 서로의 수를 보고 둠).
- **플레이어 선행** → 적 행동.
- 핵심 수싸움 = 완전정보 포지셔닝 + 슬링샷 조준 실행 리스크.

## 4. 전투
- 같은 앵커에서 조우 시 **자동 진행**.
- **선공 모델**: 착지한 쪽(공격자)이 먼저 타격.
  - **처치 성공** → 적 사망, 공격자가 **그 앵커를 차지**.
  - **처치 실패**(적 생존) → 공격자는 **원래 앵커로 되돌아감** (상호 데미지 없음).
- → 화력으로 한 방에 끝내는 것이 곧 전진·점유.

## 5. 콤보 (Push-Forward)
- **적 처치 시 추가 행동 1회** → 연쇄 가능.
- 한 번에 죽일 화력을 갖출수록 콤보가 이어짐.
- 예) '연속 베기' — 처치 시 근방 앵커로 이동하며 타격, 금빛 섬광 연출.

## 6. 승리 조건
- **적 전멸**.

## 7. 적 AI (방향)
- 앵커 노드 그래프 + **BFS 최단경로 1칸 추격**이 베이스.
- 순수 추격만으론 무한 카이팅 → **다수 적(포위)** 등으로 압박 보완.
- 세부는 개발하며 결정.

---

## 미정 / 다음에 정할 것 (우선순위순)
1. **회복·증강의 소스** — 필드 픽업은 제거됨. 현재 둘 다 비어 있음. (킬 보상으로 묶는 방향 유력)
2. **증강 획득 시점** — N턴 / N킬 / 룸 클리어 중. (N턴은 도망 파밍 구멍 주의)
3. **맵 형태** — 단일 고정판 vs 룸 순차 클리어.
4. **적 HP / 데미지 밸런스** — 2~3방 적의 추격 수싸움 비중.
5. **조준 실패 페널티** — 하드코어 유지 vs 가이드/스냅 완화.
6. **콤보 연쇄 캡** — 무한 연쇄 vs 처치당 1회.

## 차별점 (클론 회피)
"정밀 슬링샷 조준 이동" + "교대턴 완전정보 수싸움" + "선공 기반 무피해 + 처치 콤보 연쇄"의 **결합**.
Ordia는 표면 조작만 공유, 코어 루프는 추격·포지셔닝 전략으로 완전히 다름.
